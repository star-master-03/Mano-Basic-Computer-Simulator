# 💻 Mano Basic Computer 2-Pass Assembler & Simulator

2학년 1학기 컴퓨터구조 과제로 구현한 Mano Basic Computer의 16비트 아키텍처 어셈블러 컴파일러 및 명령어 사이클 시뮬레이터(에뮬레이터) 시스템입니다. 

AI(LLM)와의 페어 프로그래밍을 통해 기본 뼈대를 구축하고, 아키텍처 명세에 맞지 않는 타이밍 로직 및 예외 처리 루틴을 직접 검증하고 수정하며 프로젝트를 완성했습니다.

---

## 🛠️ 1. 구조 및 작동 원리

### ① 2-Pass 방식 번역 (`Assembler.c`)
* **Pass 1 (심볼 테이블 생성):** 소스 코드를 훑으며 `LOOP,`와 같은 레이블의 위치를 파악하고, `Symbol` 구조체 배열(`table[100]`)에 이름과 주소를 저장합니다. 줄을 읽을 때마다 `location_counter`를 1씩 증가시켜 주소를 정확히 추적합니다.
* **Pass 2 (기계어 변환):** 파일 포인터를 처음으로 되돌린 후(`rewind`), 명령어와 피연산자를 분리하여 16진수 기계어로 번역합니다. 피연산자 주소를 심볼 테이블에서 찾아 명령어 비트와 합치며, 간접 주소(`I`)가 있을 경우 최상위 비트(`0x8000`)를 가산 처리합니다.

### ② 명령어 분류 및 예외 처리
* **MRI (Memory Reference Instruction):** `ADD`, `LDA`, `STA`, `BUN`, `ISZ` 등은 `기본 Opcode + 간접 주소 비트(I) + 심볼 주소`를 비트 연산으로 조합합니다.
* **Non-MRI (Register/IO Reference):** `CLA`, `CLE`, `HLT` 등 주소가 필요 없는 명령어는 미리 정해진 16진수 고정값(예: `7001`)을 즉시 출력합니다.
* **Pseudo-Instruction:** `HEX`, `DEC` 등은 명령어가 아니라 데이터 그 자체이므로, 기계어 형식에 맞게 변환하여 메모리에 직접 할당합니다.

### ③ 하드웨어 타이밍 시뮬레이션 (`Project#1.c`)
* 실제 CPU 내부의 버스 시스템과 레지스터(`PC`, `AR`, `IR`, `AC`, `DR`, `E`)의 데이터 이동을 재현했습니다.
* 시퀀스 카운터(`SC`) 변수를 활용하여 클럭 타이밍 사이클(**T0 ~ T6**) 단위로 하드웨어 상태를 순차 제어합니다. (T0: Fetch 주소 세팅, T1: 명령어 Fetch 및 PC 증가, T2: Decode)

## 📊 2. 테스트 프로그램 구동 및 정량 검증 (`input.asm`)

시뮬레이터의 하드웨어 로직 및 런타임 정확도를 검증하기 위해, 배열 내에서 **최댓값(MAX)을 찾아내는 알고리즘 프로그램**을 어셈블리어로 작성하여 테스트를 수행했습니다.

### ① 프로그램 핵심 알고리즘
1. **초기화:** 첫 번째 데이터(A)를 `MAX` 초기값으로 설정하고, 인덱스 포인터(`PTR`)에 다음 데이터 주소를 세팅한 뒤, 루프 카운터(`I_CNT`)를 `-4(FFFC)`로 설정하여 총 4회 반복을 준비합니다.
2. **비교 로직:** 간접 참조(`LDA PTR I`)로 가져온 배열 데이터에 대해 `CMA` -> `INC` -> `ADD MAX` 과정을 수행하여 하드웨어 레벨에서 `MAX - (현재 데이터)` 뺄셈을 처리합니다.
3. **판단 및 갱신:** 연산 결과가 음수(`SNA` 조건 만족)일 경우, 즉 현재 데이터가 기존 `MAX`보다 크다고 판단되면 점프문(`BUN NEXT`)을 건너뛰고 `STA MAX`를 통해 최댓값을 새로운 데이터로 갱신합니다.
4. **루프 제어:** `ISZ PTR`로 다음 메모리를 가리키고, `ISZ I_CNT`를 통해 카운터가 0이 될 때까지 루프를 반복한 후 `HLT`를 만나 안전하게 종료됩니다.

### ② 최종 검증 결과 (`결과물.txt`)
프로그램 종료 시점의 CPU 레지스터 및 최종 메모리 상태를 추적한 결과, 사전에 설계한 **기대 결과값과 정량적으로 100% 일치함**을 확인했습니다.

| 항목 (변수) | 기대 결과값 (Expected) | 실제 출력값 (Actual) | 검증 상태 |
| :--- | :---: | :---: | :---: |
| **MAX (최댓값)** | `000F` (배열 중 가장 큰 D의 값) | **`000F`** | **PASS** ✅ |
| **PTR (마지막 포인터 주소)** | `017` (마지막 데이터 E 다음 주소) | **`017`** | **PASS** ✅ |
| **I_CNT (루프 카운터 복귀)** | `0000` (`-4`에서 4회 증가 완료) | **`0000`** | **PASS** ✅ |
